# XM-VRF Build Instructions

## Project Structure
```
XM-VRF/
├── src/                  # Source files
├── include/              # Header files
├── CMakeLists.txt        # CMake configuration
├── Makefile              # Generated by CMake
├── build/                # Build directory (generated)
│   ├── bin/              # Compiled executable
│   └── lib/              # Static libraries
└── output_files/         # Output data files (generated at runtime)
```

## Requirements
- **CMake**: >= 3.10
- **C++**: C++17 compiler
- **OpenSSL**: libssl-dev and libcrypto (for cryptographic functions)

### Installing Dependencies

#### Linux (Ubuntu/Debian)
```bash
sudo apt-get update
sudo apt-get install -y cmake g++ libssl-dev
```

#### macOS (Homebrew)
First, install Homebrew if you don't have it:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Then install dependencies:
```bash
brew install cmake openssl
```

If Homebrew is installed in non-standard location (Apple Silicon), you may need:
```bash
brew install cmake openssl
cmake .. -DOPENSSL_DIR=$(brew --prefix openssl)
```

#### Windows (with MSVC)
- Install CMake: https://cmake.org/download/
- Install Visual Studio with C++ tools
- Install OpenSSL: https://slproweb.com/products/Win32OpenSSL.html

## Building the Project

### Step 1: Clean Previous Build (Optional)
```bash
cd /path/to/XMVRF
rm -rf build output_files
```

### Step 2: Configure with CMake
```bash
mkdir -p build
cd build
cmake ..
```

### Step 3: Build the Project
```bash
make -j4
```

Or with Ninja (faster):
```bash
cmake .. -G Ninja
ninja
```

### Step 4: Run the Executable
```bash
cd /path/to/XMVRF
./build/bin/xmvrf
```

## Portable Build Design

This project is designed to be portable across different devices:

1. **CMake Configuration**: Uses `CMAKE_RUNTIME_OUTPUT_DIRECTORY` to place executables in `build/bin/`
2. **Output Files**: All generated output files are saved to `output_files/` directory
3. **No Platform-Specific Files**: Build artifacts remain in `build/` directory only
4. **Clean Build**: You can safely delete `build/` and `output_files/` directories and rebuild

## Troubleshooting

### macOS - CMake Not Found
If you get `zsh: command not found: cmake`:
```bash
brew install cmake
```

### macOS - OpenSSL Not Found
If CMake can't find OpenSSL:
```bash
# Check if OpenSSL is installed
brew list openssl

# If not installed, install it
brew install openssl

# Configure CMake with OpenSSL path
cd build
cmake .. -DOPENSSL_DIR=$(brew --prefix openssl)
make -j4
```

### macOS - Homebrew Not Installed
Install Homebrew first:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Then install dependencies:
```bash
brew install cmake openssl
```

### CMake Not Found
- Ensure CMake is installed: `cmake --version`
- Add CMake to PATH if needed

### OpenSSL Not Found
**Linux:**
```bash
sudo apt-get install libssl-dev
# Find OpenSSL location if custom installation:
brew --prefix openssl  # macOS
```

**macOS with custom OpenSSL:**
```bash
cmake .. -DOPENSSL_DIR=$(brew --prefix openssl)
```

**Windows:**
- Set environment variable: `OPENSSL_DIR=C:\Path\To\OpenSSL`

### Compilation Errors
1. Ensure C++17 is available: `g++ --version`
2. Check all source files are present in `src/` and headers in `include/`
3. Try a clean build: `rm -rf build && mkdir build && cd build && cmake .. && make`

## Cleaning the Build

```bash
# Remove all build artifacts (safe, code is preserved)
rm -rf build/

# Remove all output files (safe, code is preserved)
rm -rf output_files/

# Remove both
rm -rf build/ output_files/
```

## Output Files

When you run the executable, it generates verification and cryptographic files in the `output_files/` directory:

- `wots_signature.txt` - WOTS+ signature data
- `wots_public_key.txt` - WOTS+ public key
- `ltree_root.txt` - L-Tree root hash
- `xmss_root.txt` - XMSS tree root
- `xmss_auth_path.txt` - XMSS authentication path
- `xmvrf_output.txt` - XM-VRF output (y value)
- And other proof and signature files

You can safely delete the `output_files/` directory anytime - it will be recreated on next run.

## Verifying the Build

Test that the executable runs:
```bash
./build/bin/xmvrf << 'EOF'
7
EOF
```

This should:
1. Print the welcome message
2. Show the main menu
3. Create the `output_files/` directory
4. Exit cleanly

## Cross-Platform Notes

- **Linux/macOS**: Same build process for both
- **Windows**: Use Visual Studio CMake integration or command-line with MSVC toolchain
- **Raspberry Pi**: Install `cmake` and `libssl-dev` via apt, then follow standard build steps

## Build Optimization

For faster builds on multi-core systems:
```bash
make -j$(nproc)  # Linux/macOS
make -j%NUMBER_OF_PROCESSORS%  # Windows
```
